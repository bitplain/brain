Brain — Инициализация проекта¶

Brain — проект документации на базе Zensical, поддерживаемый организацией bitplain.

О проекте¶

Репозиторий bitplain/brain содержит базу знаний и техническую документацию, собранную с помощью генератора статических сайтов Zensical.

Структура проекта¶

brain/
├── docs/          # Исходные файлы документации (Markdown)
│   ├── index.md   # Главная страница
│   ├── init.md    # Описание проекта (этот файл)
│   └── markdown.md
├── zensical.toml  # Конфигурация Zensical
└── .github/       # CI/CD workflows

Быстрый старт¶

Команда Описание
zensical serve Запустить локальный сервер
zensical build Собрать статический сайт
zensical new Создать новый проект

Запуск из VS Code¶

В репозитории добавлены задачи VS Code для локального preview и сборки сайта.

  1. Откройте палитру команд и выберите Tasks: Run Task.
  2. Для первого запуска выполните Brain: Bootstrap Zensical.
  3. Для тестового сайта выполните Brain: Preview site.
  4. Для сборки статического сайта выполните Brain: Build site.

При работе через VS Code tunnel сервер стартует на удалённой машине, а VS Code должен автоматически пробросить порт 8000 и открыть сайт в локальном браузере.

Если автооткрытие не сработает, откройте вкладку Ports в нижней панели VS Code, найдите порт 8000 и выберите Open in Browser.

Альтернативно можно открыть вкладку Run and Debug и выбрать конфигурацию Brain: Preview site или Brain: Build site.

Задачи скачивают официальный wheel-артефакт Zensical из GitHub Releases в локальный кэш .tools/zensical, поэтому локальная установка zensical, pip, venv и Docker не требуется.

Первый bootstrap может занять несколько минут, так как Zensical загружается по сети.

Если preview-задача уже запущена, остановить её можно через Tasks: Terminate Task в VS Code.

Технологии¶

  • Zensical — генератор документации с поддержкой тем, навигации, поиска и многого другого.
  • Markdown — формат исходных файлов документации.
  • GitHub Actions — автоматизация сборки и публикации.
