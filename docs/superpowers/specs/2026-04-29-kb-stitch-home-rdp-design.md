# Дизайн: главная и RDP в стиле Stitch под Zensical

**Статус:** утверждено для плана реализации  
**Дата:** 2026-04-29  
**Контекст:** перенос внешнего вида из `cofi-knowledge-base (1)` (React) в репозиторий `brain`, где сайт собирается **Zensical** из `docs/` и конфигурации `zensical.toml`. Хаб инструкций (`docs/instructions/index.md`) уже приведён к стилю `kb-cofi`.

## Цели

1. Визуально приблизить **главную страницу** и **страницу удалённого рабочего стола** к макету React, не вводя второй фронтенд-стек.
2. Сохранить **нативное поведение** Zensical: одна навигация из `nav`, встроенный поиск, крошки (`navigation.path`), копирование кода (`content.code.copy`), без дублирующих UI в теле страницы.
3. Для RDP (**выбор A**) — **не раздувать текст** до длинной статьи из макета; меняется в основном «оболочка» и типографика, контент остаётся короткой памяткой.

## Не входит в scope

- Декоративная мета («обновлено N дней назад», «время чтения», опрос «была ли полезна») и фиктивные счётчики.
- Второе поле поиска в контенте страницы и имитация ⌘K — используется только поиск темы в шапке.
- Отдельный HTML-сайдбар «статьи раздела» дублирующий `nav` — вместо этого стилизация нативного `.md-sidebar`.
- Включение `custom_dir` / overrides шаблонов — только если после итерации CSS окончательно не хватает (отдельное решение).
- Замена базы знаний на Vite/React.

## Архитектура

| Компонент | Источник правды | Кастомизация |
|-----------|-----------------|--------------|
| Структура сайта, пункты меню | `zensical.toml` → `nav` | при добавлении страниц — только там |
| Хлебные крошки | фича `navigation.path` | `docs/stylesheets/extra.css` |
| Левое меню | тема Zensical | `extra.css` для страниц с классами вроде `kb-cofi` / `kb-doc--cofi` |
| Поиск | шапка темы | `extra.css` (при необходимости согласовать с тёмной палитрой) |
| Копирование значений | fenced code blocks + `content.code.copy` | контент в `.md` |
| Визуальные оболочки секций | классы на корневых обёртках в `.md` (как у хаба инструкций) | `extra.css` |

Поток для автора: правки в markdown и при необходимости в `nav`; визуал преимущественно в CSS.

## Главная (`docs/index.md`)

- Сохранить подход **обёртки с классами** (например расширение текущих `kb-home`, `kb-home--stitch`): тёмный фон, ambient-подсветка, hero (заголовок + лид), сетка из **двух** карточек в духе `kb-cofi__card`.
- **Рекомендуемая схема карточек:**  
  - первая — «Инструкции» → `instructions/`;  
  - вторая — **живая ссылка** на `instructions/vipnet-client-windows/` (вторая опубликованная инструкция), чтобы не оставлять неработающую заглушку.
- **Альтернатива (если потребуется визуальный паритет с макетом):** вторая карточка — неактивная «Скоро появится» без ссылки; тогда в спеке/PR явно фиксируется сознательный UX-компромисс.
- Не добавлять в разметку главной дублирующий поиск; при желании усилить «stitch»-шапку — править стили **существующего** хедера темы.

## Страница RDP (`docs/instructions/remote-desktop.md`)

- Оставить **текущий смысл и структуру** блоков (вступление, важное замечание, сравнение экранов, шаги со скриншотами, summary strip).
- Корневая обёртка остаётся в духе `kb-doc kb-doc--cofi` (или эквивалент, согласованный с существующими селекторами в `extra.css`).
- **Крошки:** не вставлять ручной `<nav>` в markdown; полагаться на `navigation.path`, доработать вид через CSS.
- **Копируемая строка:** если в тексте есть повторяемый шаблон логина или команды, оформить как **fenced code block**, чтобы использовалась нативная кнопка копирования.
- Левую колонку макета React «Статьи в этом разделе» **не воспроизводить отдельной разметкой** — это дублирование `nav`; при необходимости визуально подтянуть боковое меню темы под статьями инструкций.

## Стили (`docs/stylesheets/extra.css`)

- Добавить или расширить правила для главной (hero, сетка карточек, согласованность с `kb-cofi`).
- Для RDP — донастройка существующих классов `kb-doc--cofi` (отступы, заголовки, панели, summary strip) без ломки markdown-рендера Zensical.
- Селекторы по возможности завязаны на `:has(.kb-…)` у контейнера контента, как уже сделано для `kb-home--stitch` и `kb-cofi`, чтобы не ломать остальные страницы.

## Тестирование и приёмка

1. `python3 scripts/zensical_cli.py build --clean` завершается без ошибок.
2. Ручной проход: главная → обзор инструкций → RDP → ViPNet; проверка крошек и левого меню.
3. На RDP: скрины и вёрстка не «едут» на узкой ширине; при наличии code block — кнопка копирования работает.
4. Нет второго поля поиска в теле страниц; поиск в шапке открывается и находит страницы.

## Зависимости и риски

- Обновление Zensical может изменить классы темы; кастом ограничен `extra.css` и несколькими обёртками в markdown — объём правок при апгрейде предсказуем.
- Если палитра или шрифты темы изменятся, может понадобиться подстройка контрастов в `extra.css`.
